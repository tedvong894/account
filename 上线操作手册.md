# 小鱼记账 · 上线操作手册

> 代码已在本机初始化为 git 仓库并完成了首次提交。
> 本手册分两步：**① 推到 GitHub（含可选 GitHub Pages 在线访问）**，**② 启用 Supabase 实时同步**。
> 涉及你个人账号密码 / 密钥的操作需要在你自己的浏览器或终端里完成（本助手无法登录你的 GitHub / Supabase 账号）。

---

## ① 把代码推到 GitHub

### 方式 A：用 GitHub 网页上传（你习惯的方式，最简单）
1. 打开 https://github.com/new ，新建一个仓库，例如 `xiao-yu-jizhang`，**不要**勾选 README/.gitignore（本地已经有了）。
2. 在你电脑上打开该仓库页面 → **Add file → Upload files**，把本目录（记账软件）里的这些文件和文件夹拖进去：
   - `index.html`
   - `css/`（整个文件夹）
   - `js/`（整个文件夹）
   - `sql/schema.sql`
   - `README.md`、`上线操作手册.md`、`.gitignore`
3. 提交（Commit）。完成。

### 方式 B：用 git 命令行（如果你本地配好了 GitHub 账号）
在本目录终端执行（把 `<你的仓库地址>` 换成真实的，如 `https://github.com/tedwong/xiao-yu-jizhang.git`）：

```bash
git remote add origin <你的仓库地址>
git branch -M main
git push -u origin main
```

> 注：本助手所在环境没有你的 GitHub 写权限（连接器为只读），所以由你本机执行上面的 push 即可。

### 可选：开启 GitHub Pages（免费在线访问）
仓库推上去后：
1. 仓库页面 → **Settings → Pages**
2. Source 选 **Deploy from a branch**，Branch 选 **main** / 目录 **/ (root)**
3. Save。几分钟后会得到一个 `https://<用户名>.github.io/<仓库名>/` 的网址，手机电脑都能直接打开记账。

## ② 启用 Supabase 实时同步

代码已内置「本地存储 / Supabase」双后端自动切换。按下面 4 步，你的记账数据就会存到你自己的 Supabase、多设备实时同步。

### 第 1 步：新建 Supabase 项目
1. 打开 https://supabase.com → 登录 → **New project**。
2. 填名称（如 `xiao-yu`）、设一个数据库密码（记好）、选区域，Create。
3. 等约 1 分钟项目就绪。

### 第 2 步：建表（跑一次 SQL）
1. 项目控制台左侧 → **SQL Editor → New query**。
2. 把本仓库 `sql/schema.sql` 的**全部内容**粘贴进去，点 **Run**。
   - 这会创建 `ledgers / categories / transactions / budgets / assets` 五张表、开放匿名读写权限、并开启实时（Realtime）发布。
   - ⚠️ 已修复：`assets` 表也开启了 RLS，否则匿名角色查不到资产。

### 第 3 步：复制项目地址和密钥
1. 项目控制台左侧 → **Project Settings → API**。
2. 复制两样东西：
   - **Project URL**（形如 `https://xxxx.supabase.co`）
   - **anon public key**（很长的 `eyJ...` 字符串）

### 第 4 步：填进 App，开启实时
打开记账 App（本地或 GitHub Pages 都行）→ 底部「我的」→ **Supabase 设置**：
- 把上面复制的 URL 和 anon key 分别粘贴进去，点保存。
- 提示「已连接 Supabase ✓」即成功；顶部状态会从「本地模式」变成「实时同步中」。
- 首次连接会自动把默认分类 / 资产账户写入你的 Supabase（不会覆盖你已有的数据）。

> 之后所有记账、改分类、改资产余额都会实时存到 Supabase，换设备 / 刷新页面数据都在。

### 关于已有本地数据
上线前你在「本地模式」里试记的数据**不会自动迁移**到 Supabase（是两套存储）。两种处理：
- 直接在新账号重记（推荐，干净）；
- 或先在「我的 → 数据导出」备份（若需要我可再加导出功能），上线后再导入。
